Exclusão de Funcionários (cad_947)

Remover funcionários sem movimentação.

Está disponível no módulo /adm/cad_947 a tela que permite realizar a exclusão de funcionários sem movimentação.

A exclusão pode ser realizada utilizando filtros de Administradora, Conveniada, intervalo de códigos ou pela opção "Informar funcionários por código".

Formato para informar códigos
Exemplos:1, 1-1, 2-0
Regras de Interpretação:
  • Código sem "-x" (ex: 1): Sistema considera como 1-0 (Titular).
  • Código com "-x" (ex: 1-1): Número antes do hífen é o Titular. Número após é o cod_seqfunc do dependente.
  • Valor -0: Sempre representa o titular.
Exemplos Práticos:
  • 1 → Titular 1 (1-0)
  • 1-1 → Titular 1, Dependente 1
  • 2-0 → Titular 2
Seleção de Funcionários

É possível definir quais registros serão considerados:

  • Todos

    Considera todos os funcionários sem movimento conforme filtros.

  • Excluir somente dependentes

    Remove apenas dependentes sem movimentação.

  • Excluir somente nomes repetidos (últimos)

    Remove duplicados, mantendo o primeiro cadastro.

  • Excluir somente CPFs repetidos (últimos)

    Remove CPF duplicado, mantendo o primeiro cadastro.

  • Limpar estoque no name

    Remove registros residuais associados ao cadastro.

Obs.: Recomendamos revisar os filtros antes da exclusão, pois o processo remove definitivamente os registros que não possuem movimentação.

Habilitar Múltiplos Créditos (Conveniada)

Configuração de parâmetros e cadastro.
1
Configuração de Parâmetros Globais

Primeiro, é necessário ativar os parâmetros no nível da administradora e do grupo.

  • No Grupo de Administradora: Habilite o parâmetro LIBERARCAMPOMULTIPLOSCREDITOS
  • Na Administradora: Habilite o parâmetro FLG_BOLETOADICIONAL
2
Configuração no Cadastro

Após ajustar os parâmetros, realize a alteração na conveniada:

  1. Acesse o Cadastro de Conveniadas.
  2. Vá até a seção/aba Crédito.
  3. Localize a opção Liberar lançamento de múltiplos créditos.
  4. Altere o valor para SIM.
Nota: Certifique-se de salvar as alterações em cada etapa para que a configuração tenha efeito.

Recuperação de Senha (Pay/Util)

Procedimento via Logs no Monitoramento.
  1. Acesse o Monitoramento > Gerenciamento de Aplicações.
  2. Filtre por "Convcard Pay" e abra a aplicação "API Gateway MSG".
  3. Em outra aba, acesse o site do Pay, coloque o CNPJ e clique em "Recuperar Senha".
  4. Volte ao log, copie a URL de recuperação gerada.
  5. Acesse a URL e altere a senha para 1234.

Remessa, Histórico e Tipos de Rede.
Para gerar remessa de crédito:
  • Verificar se banco está cadastrado na tela /adm/cad_925.
  • Sequencial deve iniciar com 1 (tabela cadrbcrd).
  • Contas a pagar baixado.

Configuração Quicktext (E-mails Padrão)

Instalação da extensão e importação dos modelos de resposta para o Thunderbird.
Arquivo de Configuração

Baixe o arquivo XML com as respostas prontas antes de iniciar.

Baixar Modelos
1
Instalar a Extensão
  1. No Thunderbird, clique na Engrenagem (Configurações) (canto inferior esquerdo).
  2. Selecione "Extensões e Temas".
  3. Na barra de pesquisa, digite Quicktext.
  4. Vai abrir uma nova aba. Clique no botão verde "Adicionar ao Thunderbird" e confirme a instalação.
2
Importar os Modelos
  1. Após instalar, clique no ícone do Quicktext que apareceu no topo do Thunderbird.
  2. Na janela que abrir, vá na aba "Geral".
  3. Clique no botão Importar (lado direito).
  4. Navegue até a pasta onde você baixou o arquivo e selecione-o.
  5. Pronto! As respostas aparecerão na tela de composição de e-mail.

Cancelamento de Boleto / Permissão

Configuração do parâmetro FLG_LIBCANCELBOLCONV.

A liberação deve ser feita pela Tela de Parâmetros. O parâmetro pode ser ativado tanto para o Grupo de Conveniada quanto para a Conveniada.

Parâmetro
FLG_LIBCANCELBOLCONV
Campo 1
CONV. PERMIT. SOLICITA CANC. BOLETO

Serve para liberar ou desativar a conveniada de realizar a solicitação de cancelamento do boleto para a Administradora.

Campo 2
PERMITE CONVENIADA EXCLUIR BOLETO REGISTRADO DIRETAMENTE

Se ativo, permite que a própria conveniada realize o cancelamento do boleto de forma direta.

Atenção: A conveniada só conseguirá realizar o cancelamento se o boleto não estiver liquidado.

Parâmetros do Sistema

HABILITA_COMPROVANTE_MOVIMENTO_APP

Habilita captura de notas fiscais no App (Despesas).
Módulo Despesas Corporativas
HABILITA COMPROVANTE MOVIMENTO APP
Níveis de Aplicação
Grupo de AdministradoraAdministradoraGrupo de ConveniadaConveniada
Cenário de Uso

Ideal para colaboradores em viagens de negócios ou que realizam compras corporativas. Elimina a necessidade de guardar notas físicas para entregar posteriormente.

Como Funciona no App

Ao definir como SIM, o aplicativo libera um botão de câmera na tela de movimentação.

  • O usuário fotografa a nota fiscal/recibo na hora da compra.
  • A imagem é enviada automaticamente para o sistema.
  • O RH ou gestor financeiro visualiza o comprovante instantaneamente para conciliação.

FLG_USUARIO_CARTAO_VITUAL

Habilita/Desabilita módulo de Cartão Virtual.
FLG_USUARIO_CARTAO_VITUAL
Comportamento Padrão: ATIVADO

O cartão virtual vem habilitado nativamente. Utilize este parâmetro apenas se a empresa solicitar o bloqueio.

Para desativar: Preencher o campo valor com NÃO.

FLG_USUARIO_QR_CODE

Habilita/Desabilita vendas via QR Code.

FLG_USUARIO_QR_CODE
Bloqueio de Funcionalidade

Seguindo a lógica do cartão virtual, algumas conveniadas não autorizam o uso de QR Code. Para remover a opção do App do usuário, configure o parâmetro com valor NÃO.

FLG_NAOEXIBCREDEXP (CAD_748)

Inibir lançamento de créditos expirados.

FLG_NAOEXIBCREDEXP
Restrição Operacional

Utilizado para inibir o lançamento de créditos expirados através da tela crt/grupoadmin/cad_748. Geralmente solicitado via ticket para impedir operações retroativas indevidas.

FLG_LIBERATELAIMPOR

Libera tela de Importação Centralizada.

FLG_LIBERATELAIMPOR
Finalidade

Habilita a visualização das telas para Importação Centralizada no módulo da conveniada.

Obs.: Além do parâmetro, é necessário liberar as permissões de tela manualmente no cadastro de usuários (cad_872). Mas isso quem realiza é a própria administradora.

FLG_EXIBENUMCARTAO

Exibe numeração sem máscara (Relatórios).

FLG_EXIBENUMCARTAO
Segurança da Informação

Por padrão, os relatórios exibem o número do cartão mascarado (ex: 639321********1234) por segurança. Ativar este parâmetro remove a máscara, exibindo a numeração completa no módulo Conveniada.

FLG_CNPJDUPLO (CAD_863)

Permite duplicar o CNPJ/CPF no cadastro.

FLG_CNPJDUPLO
Permissão de Duplicidade

Ao definir como SIM, o sistema remove a trava de unicidade, permitindo cadastrar uma nova Conveniada (na tela cad_863) utilizando um CNPJ ou CPF que já existe na base de dados.

FLG_CADMULTICRED

Habilita cadastro múltiplo de credenciadas.
FLG_CADMULTICRED
Múltiplos Vínculos

Este parâmetro habilita a funcionalidade de criar múltiplos cadastros para a mesma rede de credenciadas, facilitando a gestão de filiais ou pontos de venda distintos sob a mesma administração.
